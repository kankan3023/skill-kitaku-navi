# 災害時帰宅ルートスキル 設計ドキュメント

> このドキュメントは claude.ai での設計会話をまとめたもの。
> Claude Code での開発時に参照すること。
> プロジェクト: `/home/claude/bosai-manual/` に配置想定。

---

## プロジェクト概要

AGI Lab AI Agent ハッカソン（2週間）の提出作品として、**災害時の徒歩帰宅ルートを生成し、PDF出力するClaude Code スキル**を開発する。

### コンセプト

- 「最短ルート」ではなく**「最安全ルート」＋「帰宅支援情報付き」**
- ルート上の**通過自治体ごとに防災情報を自律収集**するエージェント的振る舞い
- スマホが使えない・充電切れの状況でも**紙（PDF）で持てる**価値
- 東京都の「キタコンDX」（発災後リアルタイム用）との差別化：**事前に自分専用の帰宅計画を作成**

### 制約

- **APIキー不要**のサービスのみ使用（ハッカソン提出先で動作する必要あり）
- 対象地域は**東京都内**に限定
- Claude Code スキルとして実装（SKILL.md ベース）
- 「スキルは育てるもの」：最小構成から段階的に機能追加

## 開発ステップ

### Step 1（MVP・2〜3日）
- 出発地・目的地の住所入力
- 国土地理院ジオコーディングで座標変換
- OSRM で徒歩ルート取得
- 距離・推定所要時間を計算
- PDF に地図画像とルート情報を出力

### Step 2（実用性アップ・3〜4日）
- ルート沿いの施設情報を Overpass API で取得（コンビニ・公衆トイレ等）
- 東京都オープンデータから避難所・給水拠点をプロット
- 距離に応じた必要水分量・休憩ポイントの自動提案

### Step 3（差別化 - エージェント的振る舞い・3〜4日）
- **ルートの座標列から通過自治体を自動特定**（逆ジオコーディング）
- 各自治体のオープンデータを自動収集（自治体標準オープンデータセット活用）
- ハザードマップタイル重畳でルート上の危険エリア警告
- 災害種別（洪水・地震）によるルート比較

### Step 4（磨き・残り日数）
- 家族構成ヒアリングによるパーソナライズ
- 発災後タイムライン型の行動指針
- PDF全体のデザイン仕上げ
- 自治体ごとのセクション分けページ構成

## 使用するデータソース（全てAPIキー不要）

### ルート・地図基盤

| サービス | 用途 | エンドポイント例 |
|---|---|---|
| 国土地理院 ジオコーディング API | 住所 → 座標変換 | `https://msearch.gsi.go.jp/address-search/AddressSearch?q={住所}` |
| 国土地理院 タイル | 地図画像の背景 | `https://cyberjapandata.gsi.go.jp/xyz/std/{z}/{x}/{y}.png` |
| OSRM (公開デモサーバー) | 徒歩ルート計算 | `https://router.project-osrm.org/route/v1/foot/{lng1},{lat1};{lng2},{lat2}?overview=full&geometries=geojson` |
| Overpass API | OSM データ検索（コンビニ・トイレ等） | `https://overpass-api.de/api/interpreter` |

### 東京都オープンデータ（都レベル）

| データ | 形式 | URL |
|---|---|---|
| 避難所一覧 | CSV | `https://www.opendata.metro.tokyo.lg.jp/soumu/130001_evacuation_facility.csv` |
| 避難場所一覧 | CSV | `https://www.opendata.metro.tokyo.lg.jp/soumu/130001_evacuation_area.csv` |
| 災害時給水ステーション（213箇所） | CSV | 東京都オープンデータカタログ `catalog.data.metro.tokyo.lg.jp` |
| 帰宅支援対象道路 | 東京都防災マップ上 | `map.bosai.metro.tokyo.lg.jp` |

### 自治体別オープンデータ（標準データセット）

デジタル庁「自治体標準オープンデータセット」に基づき、各区市町村が共通フォーマットで公開：

- **指定緊急避難所一覧**（災害種別対応あり）
- **AED設置箇所一覧**
- **医療機関一覧**
- **公衆トイレ一覧**
- **防災行政無線設置一覧**
- **消防水利施設一覧**
- **公衆無線LAN AP一覧**

> フォーマットが統一されているため、自治体が変わってもパースロジックの変更が不要。

### AED 専用

| サービス | 用途 | エンドポイント例 |
|---|---|---|
| AED オープンデータ API | 最寄りAED検索 | `https://aed.azure-mobile.net/api/AEDSearch?lat={緯度}&lng={経度}&r={半径m}` |

### ハザードマップ

| データ | 配信元 |
|---|---|
| 洪水浸水想定区域 | ハザードマップポータル（国土交通省）タイル配信 |
| 高潮浸水想定区域 | 東京都港湾局オープンデータ |
| 土砂災害警戒区域 | ハザードマップポータル タイル配信 |

## エージェント的処理フロー

```
1. ユーザーとの対話で出発地・目的地をヒアリング
2. 国土地理院 API で座標変換
3. OSRM で徒歩ルート計算
4. ルートの座標列から通過自治体を特定（逆ジオコーディング）
   例: 新宿区 → 中野区 → 杉並区 → 武蔵野市 → 小金井市
5. 各自治体ごとに自動で情報収集:
   - 避難所・避難場所（都CSV or 区の標準データ）
   - AED（オープンデータAPI）
   - 給水拠点（都水道局CSV）
   - 公衆トイレ（区の標準データ）
   - 医療機関（区の標準データ）
6. ルートから一定距離内（例: 500m）の施設を抽出
7. 自治体ごとにセクション分けしてPDF出力
```

## 帰宅支援ステーション情報

東京都が指定する災害時帰宅支援ステーション（水道水・トイレ・災害情報を提供）：

- 全都立学校
- 東京武道館
- コンビニ（セブン、ファミマ、ローソン、ミニストップ、ポプラ）
- 飲食チェーン（吉野家、ドトール、タリーズ、リンガーハット等）
- ガソリンスタンド
- カーディーラー（日産、トヨタ、マツダ系販売店）
- その他（オートバックス、農協等）

> Overpass API で amenity=convenience 等を検索すれば、ルート沿いの帰宅支援ステーション候補を自動取得可能。

## 技術メモ

- **地図画像生成**: Python の folium または staticmap ライブラリで静的地図画像を生成し、PDF に埋め込む
- **PDF生成**: 既存の bosai-manual スキルの PDF 生成ロジックを拡張
- **座標→自治体の判定**: 国土地理院の逆ジオコーディング、またはポリゴンデータ（国土数値情報の行政区域データ）を使用
- **Overpass クエリ例**:
  ```
  [out:json][timeout:25];
  (
    node["amenity"="drinking_water"](around:500,{lat},{lng});
    node["amenity"="toilets"](around:500,{lat},{lng});
    node["amenity"="convenience"](around:300,{lat},{lng});
  );
  out body;
  ```
